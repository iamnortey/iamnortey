## Repository Overview

This is **iamnortey/iamnortey** - a special GitHub profile README repository. The `README.md` file in this repository is displayed on the owner's GitHub profile page at https://github.com/iamnortey.

## Repository Structure

```
iamnortey/
```

## Purpose

This repository serves as a personal GitHub profile landing page. It allows customization of what visitors see when they view the GitHub profile.

## Development Guidelines

### Working with the README.md

1. **Content**: The README.md supports GitHub-flavored Markdown, including:
   - Headings, lists, and text formatting
   - Images and GIFs
   - Links and badges
   - HTML elements (limited subset)
   - Emoji shortcodes

2. **Best Practices**:
   - Keep content concise and scannable
   - Use visual elements (badges, stats, icons) to enhance readability
   - Include relevant links to projects, social profiles, or contact information
   - Update regularly to reflect current work and interests

### Common Enhancements

Profile READMEs often include:
- Introduction/bio section
- Skills and technologies (often with badges/icons)
- GitHub statistics widgets
- Featured projects
- Contact information
- Social media links
- Current focus or learning goals

## AI Assistant Instructions

When assisting with this repository:

1. **Profile Updates**: Help craft professional, engaging profile content that reflects the user's skills, projects, and personality

2. **Markdown Formatting**: Ensure proper GitHub-flavored Markdown syntax

3. **Visual Elements**: Suggest appropriate badges, stats widgets, or visual enhancements when requested

4. **Consistency**: Maintain consistent formatting and style throughout the README

5. **Testing**: Remember that changes to README.md are immediately visible on the GitHub profile after pushing to main
